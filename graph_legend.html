Trang này giải nghĩa các biểu đồ được biên soạn bởi doxygen.

Hãy xem xét các ví dụ sau:

/*! Các lướp không thấy được bởi sự cắt ngắn */
Lớp không nhìn thấy class { };
/*! class bị cắt, quan hệ kế thừa bị ẩn */
class bị cắt : bị ẩn toàn cục { };
/* Class không được mô tả với các chú giải doxygen */
class không được mô tả { };
/*! Class được kế thừa sử dụng các kế thừa public */
class PublicBase : public Truncated { };
/*! template class */
template<class T> class Templ { };
/*! Class được kế thừa sử dụng kế thừa protected */
class ProtectedBase { };
/*! Class được kế thừa sử dụng kế thừa protected private */
class PrivateBase { };
/*! Class được sử dụng bởi các class kế thừa */
class được sử dụng { };
/*! Super class kế thừa một số các class khác */
class được kế thừa : public PublicBase,
protected ProtectedBase,
private PrivateBase,
public Undocumented,
public Templ<int>
{
private:
Used *m_usedClass;
};

Kết quả trong biểu đồ sau đây:

Các hộp trong biểu đồ trên có ý nghĩa như sau:

  • Hộp màu xám biểu thị cấu trúc clas cho biểu đồ được thể hiện.
  • Hộp có khung mầu đen biểu thị struct hoặc class được mô tả.
  • Hộp có khung mầu xám biểu thị struct hoặc class chưa được mô tả.
  • Hộp có khung mầu đỏ biểu thị struct hoặc class được mổ tả khi mà không phải tất cả các quan hệ kế thừa/containment được biển diễn.. Biểu đồ bị cắt nếu nó không nằm trọn trong các biên được cho trước..

Các mũi tên có ý nghĩa như sau::

  • Mũi tên mầu xanh đậm biểu thị các quan hệ kế thừa công khai giữa 2 class.
  • Mũi tên màu xanh lá cây đậm biểu thị kế thừa được bảo về (protected).
  • Mũi tên đỏ đậm biểu thị kế thừa dạng private.
  • Mũi tên màu hồng đứt quảng biểu thị class được sử dụng bởi class khác. Mũi tên được gán nhãn với các giá trị mà các calsss hoặc struct được truy cập tới.
  • Mũi tên vàng đắt quãng được thị quan hệ giữa template instance và template class được dẫn xuất từ đó. Mũi tên được gán nhãn với tham số của template.
